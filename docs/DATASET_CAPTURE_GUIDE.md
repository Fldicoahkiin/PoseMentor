# 数据规格对比与采集交付规范

## 1. 拍摄团队只需要做的事（必须）

以下是前期必须执行的最小要求，不需要技术背景。

| 优先级 | 必做项 | 最低标准 |
|---|---|---|
| P0 | 机位固定 | 四机位固定为 front/left/right/back，全程不挪动 |
| P0 | 画质统一 | 四路都用 1080p，建议 60 FPS（最低 30 FPS） |
| P0 | 同步动作 | 每段开始先拍手 1 次（全部机位都拍到） |
| P0 | 人体完整入镜 | 全身不出画，脚和手都不能裁切 |
| P1 | 动作段长度 | 单段 8~20 秒，前后各留约 2 秒静止 |
| P1 | 文件命名一致 | `session_xxx/front.mp4` `left.mp4` `right.mp4` `back.mp4` |
| P1 | 机位变化留记录 | 机位一旦移动，必须标注“新 session” |

## 2. 机位布局建议（四机位）

当前系统要求的 `front / left / right / back` 是固定命名约定。
要让后续三角化稳定，机位布局建议如下：

| 机位 | 推荐方位 | 说明 |
|---|---|---|
| `front` | 0° | 正前方主视角 |
| `left` | 90° | 左侧完整侧视 |
| `back` | 180° | 背面补遮挡 |
| `right` | 270° | 右侧完整侧视 |

补充要求：

- 四台设备尽量围绕拍摄区域均匀分布，不要都集中在前方
- 四台高度尽量一致，建议胸口到肩部高度
- 焦段固定，不要中途切换 0.5x / 1x / 2x
- 机位一旦移动，需要记为新 session，不能继续复用旧标定参数

## 3. 我们后处理自动做的事（已实现）

拍摄团队不需要负责以下技术步骤：

- 四路视频时序粗对齐（起始偏移估计）
- 统一帧率、统一分辨率导出
- YOLO11-Pose 2D 关键点提取（单视角/四机位目录都支持）
- 对齐结果统计与可视化报告
- 训练输入格式化（`*.npz`）

当前对应脚本：

- `prepare_multiview_dataset.py`
- `extract_pose_yolo11.py --recursive`
- `visualize_multiview_report.py`

## 4. 后续自动化（规划中）

这些属于后期平台能力，不作为拍摄侧前置要求：

- 自动相机标定参数求解与版本管理
- 多机位三角化生成 3D 真值
- 三角化质量评分与异常样本自动剔除
- 自动入库校验（命名、时长、缺帧、质量阈值）

建议采用固定场地方案：机位、三脚架高度、焦段、分辨率固定后，只做一次标定，后续同场地重复采集直接复用参数。

## 5. 手机采集可用范围与精度预期

手机四机位可以用于动作标准评估，但精度取决于同步、标定、遮挡和画质，不取决于“是否专业相机”这一项。

建议按以下目标评估：

| 条件 | 3D 关节误差预期 | 说明 |
|---|---|---|
| 机位固定、同步稳定、已标定 | 约 3~6 cm | 可用于教学评分和动作对比 |
| 普通教室、手机拍摄、遮挡可控 | 约 5~10 cm | 更接近实际落地场景 |
| 同步差、遮挡重、机位不稳 | 10 cm 以上 | 结果波动大，不建议直接入库 |

影响误差的主要因素：

- 同步误差
- 标定误差
- 分辨率和快门稳定性
- 人体关键点遮挡
- 镜头畸变和自动对焦漂移

## 6. 公共数据与自采数据关键差异（简版）

| 维度 | AIST++（公共） | 自采四机位（目标） | 主要风险 |
|---|---|---|---|
| 机位 | 9 机位固定场地 | 4 机位现场采集 | 角度覆盖不足、遮挡变多 |
| 帧率/分辨率 | 约 60 FPS / 1080p | 目标 60 FPS / 1080p | 参数不一致导致训练波动 |
| 时间同步 | 有统一时间戳 | 依赖拍手同步和后处理 | 长序列漂移 |
| 相机参数 | 内外参齐全 | 初期通常缺失 | 无法直接三角化 |
| 标签 | 2D+3D 完整 | 初期通常只有视频+2D | 监督信号不足 |

## 7. AIST++ 机位与参数（项目实测）

基于当前目录 `data/raw/aistpp/annotations/aist_plusplus_final`：

- 2D 文件：`1510`
- 3D 文件：`1408`
- 2D 结构：`keypoints2d [9, T, 17, 3]`
- 3D 结构：`keypoints3d [T, 17, 3]`
- 帧率：约 `59.9988 FPS`
- 分辨率：`1920x1080`
- 相机：`c01 ~ c09`（环绕式机位组）

## 8. 交付格式（拍摄侧）

目录：

```text
data/raw/multiview/
  session_001/
    front.mp4
    left.mp4
    right.mp4
    back.mp4
    metadata.json
```

`metadata.json` 最小字段：

- `session_id`
- `action_id`
- `performer_id`
- `take_id`
- `fps`
- `resolution`

## 9. 验收红线（不达标直接重拍）

- 任一机位人物关键部位长期出画
- 四路分辨率或帧率不一致
- 缺少同步拍手片段
- 文件命名不符合约定，导致四路无法配对
