# Demo阶段精度与实时性优化（重点）

## 精度提升与难点处理（>=8条）

1. **先固定舞种模板**：演示时先限定 `gBR` 或 `gHO`，可明显降低 DTW 对齐误差。  
2. **帧率对齐**：AIST++ 与摄像头 FPS 不一致时，先重采样到统一 FPS，再做 DTW。  
3. **2D关键点平滑**：对 YOLO11 输出使用 Kalman（已内置），减少抖动引发的 3D 放大误差。  
4. **根节点对齐**：3D 比较前做髋部中心化，减少平移误差干扰 MPJPE。  
5. **关节置信度加权损失**：训练时按 2D conf 加权，降低低质量帧污染。  
6. **速度损失约束**：加入 velocity loss（已内置）抑制时序跳变。  
7. **先小样本快收敛**：先用 50~120 序列训练验证链路，再扩全量数据。  
8. **阈值分层**：关节高亮阈值、语音触发阈值、总分映射阈值分开调，避免“过度报警”。  
9. **YOLO输入分辨率折中**：640 是实时和精度平衡点，1080p 常导致 FPS 降太多。  
10. **镜头机位固定**：单摄像头演示时尽量正前方或45°固定机位，减少深度歧义。

## 常见问题

- **Q: MPJPE 显示特别大（>200mm）**  
  A: 先检查单位是否米/毫米混用，再检查训练与推理归一化参数是否一致。

- **Q: 摄像头流式会卡顿**  
  A: 降低分辨率到 640x480，减少 `seq_len` 到 49，或切换 `yolo11n-pose`。

- **Q: 语音不播放**  
  A: `edge-tts/gTTS` 依赖网络；若网络受限，先只展示文字建议。

- **Q: 没检测到人**  
  A: 增强光照，完整入镜，调低 `det_conf` 到 0.25~0.30。
